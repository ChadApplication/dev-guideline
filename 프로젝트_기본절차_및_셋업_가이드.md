## 1. 개발환경 전제조건

### 1.1 필수 소프트웨어

| 도구 | 버전 | 용도 | 설치 방법 |
|------|------|------|-----------|
| Python | 3.12.8 (pyenv global) | 백엔드 개발 | `pyenv install 3.12.8` |
| Node.js | 22 LTS | 프론트엔드 개발 | `brew install node` |
| Git | latest | 버전 관리 | `brew install git` |
| pyenv | latest | Python 버전 관리 | `brew install pyenv` |
| uv | 0.6+ | 빠른 pip 대안 (선택) | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |

### 1.2 Python 환경 관리 원칙

```
pyenv (개발용)           Homebrew (CLI 앱용)
├── 3.12.8 (global)      ├── python@3.14 (자동)
├── 프로젝트별 venv       └── python@3.13 (자동)
└── 경량 기본 환경
```

- pyenv가 개발 Python을 관리하고, Homebrew Python은 brew 앱의 자동 의존성
- 절대 brew Python을 개발에 사용하지 않음
- 프로젝트마다 반드시 venv 생성 (글로벌 환경에 패키지 설치 최소화)

## 2. 새 프로젝트 생성 절차

### Step 1: 템플릿 복사

```bash
cd ~/Projects
cp -r _template_latest my_new_project
cd my_new_project
```

### Step 2: 프로젝트 설정 변경

**run.sh 수정** (필수):
```bash
# 8행: 프로젝트 이름 변경
PROJECT_NAME="MyNewProject"

# 26-27행: 기본 포트 변경 (다른 프로젝트와 충돌 방지)
DEFAULT_BACKEND_PORT=8010
DEFAULT_FRONTEND_PORT=3010
```

**backend/.env 생성**:
```bash
cp backend/.env.example backend/.env
# 필요한 API 키 등 설정
```

**frontend/.env.local 생성**:
```bash
echo "NEXT_PUBLIC_BACKEND_PORT=8010" > frontend/.env.local
```

### Step 3: 백엔드 셋업

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### Step 4: 프론트엔드 셋업

```bash
cd frontend
npm install
```

### Step 5: Git 초기화

```bash
cd ~/Projects/my_new_project
rm -rf .git frontend/.git    # 템플릿의 git 히스토리 제거
git init
git add .
git commit -m "Initial project setup from template"
```

### Step 6: 실행 확인

```bash
./run.sh start
# Backend API: http://localhost:8010
# Frontend UI: http://localhost:3010
```

## 5. 개발 워크플로우

### 5.1 일상 개발 사이클

```
1. 서버 시작       ./run.sh start
2. 코드 수정       backend/ 또는 frontend/src/ 편집
3. 자동 리로드     FastAPI(reload=True), Next.js(HMR) 자동 반영
4. 테스트          브라우저에서 확인
5. 서버 종료       ./run.sh stop
```

### 5.2 pm.sh를 이용한 멀티 프로젝트 관리

```bash
cd ~/Projects
./pm.sh
# TUI에서 프로젝트 선택 → Start/Stop/Restart/Live Logs
```

- 각 프로젝트는 자동으로 충돌 없는 포트를 할당받음
- `.run_ports` 파일에 현재 포트 기록
- `Stop ALL Projects`로 일괄 종료 가능

### 5.3 새 API 엔드포인트 추가

**백엔드 (FastAPI)**:

1. `backend/services/` 에 서비스 로직 작성
2. `backend/app/routers/` 에 라우터 모듈 생성
3. `backend/main.py` 에서 라우터 등록: `app.include_router(my_router.router)`

**프론트엔드 (Next.js)**:

1. `frontend/src/services/` 에 API 호출 함수 작성
2. `frontend/src/components/` 에 UI 컴포넌트 생성
3. `frontend/src/app/` 에 페이지 라우트 추가

### 5.4 환경변수 관리

| 파일 | 용도 | Git |
|------|------|-----|
| `.env.example` | 템플릿 (키 이름만, 값 비움) | 포함 |
| `.env` | 실제 값 (API 키 등) | **제외** |
| `.env.local` | 로컬 오버라이드 | **제외** |
| `.env.production` | 운영 환경 설정 | 상황에 따라 |

## 7. run.sh 제어 스크립트

### 명령어

| 명령 | 동작 |
|------|------|
| `./run.sh start` | 백엔드 + 프론트엔드 백그라운드 시작 (자동 포트 할당) |
| `./run.sh stop` | 모든 서버 종료, 포트 해제 |
| `./run.sh restart` | stop → start |
| `./run.sh status` | 현재 서버 상태 확인 |
| `./run.sh live` | 서버 시작 + 실시간 로그 스트리밍 |

### 포트 관리

- 기본 포트가 사용 중이면 자동으로 다음 포트를 찾음
- `~/Projects/.allocated_ports` 에 글로벌 포트 레지스트리 유지
- 파일 잠금(`mkdir`)으로 동시 실행 충돌 방지

### macOS 호환성 주의

- `#!/bin/bash` 사용 시 macOS 기본 bash 3.2 적용
- `${VAR,,}` (bash 4+ lowercase) 대신 `$(echo "$VAR" | tr '[:upper:]' '[:lower:]')` 사용
- `sed -i` 대신 `sed -i.bak` 사용 (BSD sed)

## 10. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `bad substitution` | bash 4+ 문법을 macOS bash 3.2에서 실행 | `tr` 등 POSIX 호환 방식 사용 |
| 포트 충돌 | 이전 서버가 완전히 종료되지 않음 | `./run.sh stop` 후 `lsof -ti:PORT \| xargs kill -9` |
| venv 활성화 안 됨 | Python 경로 변경 후 venv 재생성 필요 | `rm -rf venv && python -m venv venv` |
| `No module named` | venv 미활성화 상태에서 실행 | `source venv/bin/activate` 또는 `./venv/bin/python` |
| Next.js dev 잠금 | `.next/dev/lock` 파일 잔존 | `rm -f frontend/.next/dev/lock` |
| npm 의존성 오류 | node_modules 손상 | `rm -rf node_modules package-lock.json && npm install` |
